{ Drag & Draw }

Code Challenge: Dibujo vectorial 2D con p5.js
💀 Deadline: Martes 29 de Abril del 2025 - Final del día

description()

Desarrolla una aplicación web interactiva con p5.js que permita a los usuarios dibujar, editar, borrar trazos y más funcionalidades sobre un <canvas>.

requirements()

  • Utiliza clases para representar los trazos. [Programación Orientada a Objetos]
  • Emplea arreglos para almacenar objetos, variables o instancias.
    W3Schools: Arrays
  • Implementa eventos de mouse y pantallas táctiles para la interacción del usuario.
    mouse, keyboard, touch
  • Asegura que el <canvas> y el diseño general sean responsive.
  • Utiliza un HTML semántico y accesible.

methods()

Implementa al menos cuatro funcionalidades clave para enriquecer la experiencia de usuario. Puedes elegir entre las siguientes opciones o proponer nuevas:

  • Botones o atajos de teclado para:
    • Deshacer [Ctrl+Z]
    • Rehacer [Ctrl+Y]
    • Limpiar el lienzo [Ctrl+Del]
  • Selector de color para el pincel mediante:
    • <input type="color">

      Documentación de input color
      Ejemplos en Codepen
  • Selector de grosor del pincel mediante:
    • <input type="range">

      Documentación de input range
      Ejemplos en Codepen
  • Botón para exportar el dibujo como imagen (.png)
    @see: saveCanvas()
  • Botón para exportar el dibujo en formato vectorial (.svg)

interface()

  • Utiliza <button> para ejecutar acciones en JavaScript.
  • Utiliza <input> para la entrada de datos del usuario.
  • Mantén la interfaz simple e intuitiva para facilitar su uso.
  • Asegura un diseño responsive que se adapte a diferentes dispositivos.
  • Incluye una barra de herramientas con botones grandes y accesibles.
  • Utiliza combinaciones de colores con buen contraste para mejorar la accesibilidad.
  • Selecciona una tipografía legible y con estilo coherente.

tips()

  • Utiliza como punto de partida el ejercicio desarrollado en clase Crayon
  • Si una funcionalidad te parece compleja, divídela en partes más pequeñas y utiliza pseudocódigo.
  • Organiza tu código encapsulando funcionalidades en clases.
  • Prueba tu aplicación en dispositivos móviles para verificar el soporte táctil.
  • Implementa atajos de teclado usando keyPressed().

evaluation()

  • Funcionalidad general y experiencia de usuario.
  • Calidad del código: estructura, indentación y nomenclatura.
  • Organización del proyecto en carpetas con módulos y clases bien definidos.
  • Diseño, responsividad y accesibilidad:
    • Uso adecuado del tamaño de tipografía.
    • Tamaño recomendado para tamaño de botones en múltiples dispositivos.
    • Contraste suficiente entre colores.
      Verificador de contraste
  • Presencia de metadata y uso de etiquetas semánticas.
  • Originalidad, complejidad y funcionalidad del código entregado.
  • Repositorio y commits en github.
  • Publicación en github pages.
